# Agile User Stories

This document translates Functional Requirements (FR) into Agile User Stories.

## US-01: User Registration
**As an** unregistered customer,
**I want to** register an account using my email and password,
**So that** I can securely book movie tickets.
- **Traceability**: Maps to FR-01.1

## US-02: User Login
**As a** registered customer,
**I want to** log in with my credentials,
**So that** I can receive an access token to perform secure actions.
- **Traceability**: Maps to FR-01.2, FR-01.3

## US-03: Browse Movies
**As a** customer,
**I want to** view a list of all currently available movies,
**So that** I can decide which movie I want to book.
- **Traceability**: Maps to FR-02.1, FR-02.2

## US-04: Create Booking
**As an** authenticated customer,
**I want to** book a ticket for a specific movie,
**So that** I can reserve a seat for the showtime.
- **Traceability**: Maps to FR-03.1, FR-03.2

## US-05: View Booking History
**As an** authenticated customer,
**I want to** view a list of all my previous and active bookings,
**So that** I can keep track of my movie schedule and past visits.
- **Traceability**: Maps to FR-03.3

## US-06: Cancel Booking
**As an** authenticated customer,
**I want to** cancel an existing booking I made,
**So that** I am not charged or holding a seat if my plans change.
- **Traceability**: Maps to FR-03.4, FR-03.5
